Patient Registration

Form with detailed validation documentation

Validation Flow Overview

This documentation explains the complete validation system implemented in this form.

User enters data in form fields
oninput event triggers validation function
Validation function checks field requirements
Appropriate CSS classes applied (valid/invalid)
Feedback message displayed to user
Form submission button enabled/disabled based on validity

Validation Functions

validateFirstName()
Validates the first name field for patient registration.
// Checks: Non-empty, minimum 2 characters, no numbers/special chars
// Returns: true if valid, false otherwise
// Updates: UI element styling and error message
validateLastName()
Validates the last name field for patient registration.
// Checks: Non-empty, minimum 2 characters, no numbers/special chars
// Returns: true if valid, false otherwise
// Updates: UI element styling and error message
validateRegistrationEmail()
Validates the email address using regex pattern matching.
// Pattern: /^[^\s@]+@[^\s@]+\.[^\s@]+$/
// Checks: Valid email format with domain
// Returns: true if valid, false otherwise
// Updates: UI element styling and error message
validateRegistrationPhone()
Validates phone number for Indian format (+91 XXXXXXXXXX or 10 digits starting with 6-9).
// Pattern: /^(\+91[\-\s]?)?[6-9][0-9]{9}$/
// Checks: Valid Indian phone number format
// Returns: true if valid, false otherwise
// Updates: UI element styling and error message

Validation Rules Reference

First/Last Name Requirements:
  • Minimum 2 characters
  • Maximum 50 characters
  • Letters and spaces only
  • No numbers or special characters
Email Requirements:
  • Must contain @ symbol
  • Must have valid domain
  • Must have Top Level Domain (.com, .in, etc.)
  • No whitespace allowed
Phone Number Requirements:
  • 10 digits for validation
  • First digit must be 6-9 (Indian format)
  • Optional +91 country code prefix
  • Can contain hyphens or spaces as separators

CSS Classes Used

.valid - Applied when field passes validation
border-color: #51cf66;
box-shadow: 0 0 10px rgba(81, 207, 102, 0.3);
.invalid - Applied when field fails validation
border-color: #ff6b6b;
box-shadow: 0 0 10px rgba(255, 107, 107, 0.3);

Error Messages

Error messages are displayed dynamically based on validation failure reasons. Each field has a dedicated message element (e.g., #firstNameMsg) that shows specific error text to guide user correction.

Best Practices Demonstrated

  • Real-time Validation: oninput events provide immediate feedback
  • Clear Error Messages: Users know exactly what's wrong
  • Visual Feedback: Border colors and shadows indicate status
  • Form State Management: Submit button disabled until valid
  • Code Documentation: Comments explain each validation rule
  • Modular Functions: Each field has dedicated validation function
